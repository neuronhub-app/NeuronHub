## Monitoring Progress


### Projects

This is your main page for monitoring progress.

Projects allow to group `Milestones` with ETAs into a single goal.

`Favorite` <LinkExt href="https://linear.app/probablygood/project/go-live-38f83acd2475/overview">the current Project</LinkExt>.


Estimates on issues allow to view the Progress Chart:

<ImageWithDialog maxH="60" src="https://s3.stage.neuronhub.app/docs/linear/progress-chart.avif" />


### Search

Linear search isn't usable beyond the issue titles.

Instead use <Kbd keys="Meta J" /> to open `Ask Linear` - and ask for complex queries re user activity, issue descriptions, comments, etc. Examples:
- What code has @viktor been writing in last 3 days?
- Is there an issue tracking the UTM parameters?
- What are the latest comments re "Part-Time" filters?


## Managing Specification

Linear is the single source of truth for the specification.


### Only the Issue Description is specification

Mobile/Slack/Email messages get marked as "read" by accident, and important details get buried.
Sending specs through those channels is fine as a starting point - just make sure to capture them
in the Issue Description so nothing is lost.

The same applies to Linear Comments: consolidate key decisions into the `Description`, since long comment threads lose visibility quickly.

Aim to mark Comments and Threads as `Resolved` when you move them to the `Description` - this helps others to find missed Comments later.


### Documents

All Google Documents will be moved to Linear Documents (copy-paste will import all formatting),
that support same comments and collaborative editing.
And they will be attached to their Issue or Project.

<ImageWithDialog src="https://s3.stage.neuronhub.app/docs/linear/document-comments.avif" />


### Issue detail

To keep the history readable:
- Use `Collapsible Section` in the text editor to collapse screenshots, or reduce their size.
- Mark your Comment Threads as `Resolved` to collapse them. Or post a Comment in the format `**Resolved**: {reason}`.

<ImageWithDialog src="https://s3.stage.neuronhub.app/docs/linear/resolve-thread-with-comment.avif" />
<ImageWithDialog src="https://s3.stage.neuronhub.app/docs/linear/resolve-thread-with-comment-applied.avif" />

Explore the `Issue version history`.


### Issue Relations

Whenever you transfer part of a Comment or Issue Description to another Issue - mention
the new Issue by `@{id}` - it'll be automatically `Marked as - Related to`.

Use <Kbd keys="Meta K" /> to mark it as:
- `Duplicate of {id}`
- `Sub-issue of {id}`
- etc


### Usage

Linear is built around keyboard shortcuts - using them is essential for an efficient workflow.

Use <Kbd keys="Meta K" /> for *everything*: find/create Documents, Issues, Projects, etc.

Use <Kbd keys="Meta ?" /> to view the shortcuts reference.

Use <Kbd keys="Meta Z" /> to undo *anything*, and <Kbd keys="Meta Shift Z" /> to redo.

Use `All Issues` as your entry point:
- Use its shortcut - <Kbd keys="G E Enter" />.
- Add it to your `Favorites` in Linear.
- Add it to your browser's Bookmarks Bar - always keep it one click away.


#### Issue list

Press any arrow key to enter the keyboard navigation mode:
- <Kbd keys="Shift D" /> - set `Due date` => type words `3d`, `1month`, `1w`  => <Kbd keys="Enter" />.
- <Kbd keys="Shift P" /> - set `Project`.
- <Kbd keys="P" /> - set `Priority`.
- <Kbd keys="Alt ↑/↓" /> to re-order the focused Issue, and auto-change its `Priority`.

Use <Kbd keys="Meta K" /> for all actions:
- Edit anything.
- Jump to any Issue, Thread, Comment.

Right-click on Issue -> start typing to enter the `Filter` mode.

### Mobile App

Install <LinkExt href="https://linear.app/mobile">the mobile app</LinkExt> on <LinkExt href="https://apps.apple.com/us/app/linear-mobile/id1645587184">iOS</LinkExt>
or <LinkExt href="https://play.google.com/store/apps/details?id=app.linear">Android</LinkExt>.


Consider setting <LinkExt href="https://linear.app/settings/account/notifications/mobile">Mobile Notifications</LinkExt>
to `Mentions` only. For everything else, email or `Inbox` works better.


### Emoji

Reserve emojis for Comment content and reactions - don't use them in Documents, Projects, or other structured fields,
as they don't work well as pictograms.
