## Issue Reports

I recommend triaging the <Term id="backend" /> issues daily, and the <Term id="frontend" /> once every few weeks.

The <LinkExt href="https://sentry.io/issues/alerts/rules/">default Alerts settings</LinkExt> are configured by project:
- Uptime - immediate email.
- <Term id="backend" /> - immediate email.
- <Term id="frontend" /> - if it affects 3+ users per month.

## Feedback

On error, a user is presented with a low-key popup to submit the Feedback report form.

<LinkExt href="https://sentry.io/issues/feedback/">The reports</LinkExt> can be viewed for 90 days.

Each sends an email alert. You can also setup Slack or SMS.

## Logs

All logs from the <Term id="backend" />, <Term id="frontend" />, and <Term id="worker" /> are sent to
<LinkExt href="https://sentry.io/explore/logs/">the Sentry Logs dashboard</LinkExt>.

As a non-developer you shouldn't need them, but if you want to double-check any background processes or
find out why a user encountered an issue - that's a good place to quickly check.

You can use complex filters to either search or aggregate the logs data:

<ImageWithDialog src="https://media.neuronhub.app/docs/sentry/logs.avif" alt="Logs" isDimmed />

## Uptime

Sentry monitors the uptime of your <Term id="backend" /> and <Term id="frontend" /> every 5 minutes,
and logs downtime if the response took more than 10 seconds. You'll receive email alerts if it detects any downtime.

You can view the statistics in the <LinkExt href="https://sentry.io/insights/uptime/">uptime dashboard</LinkExt>.

<ImageWithDialog src="https://media.neuronhub.app/docs/sentry/uptime.avif" alt="Uptime" isDimmed />

## Scheduled Tasks Monitor

Similar to Uptime, Sentry monitors each run of your scheduled tasks (`crons`), eg hourly <PageLink id="job-alert-emails" />,
and sends you email alerts if any task run failed to report its successful execution within 5 minutes of its start.
