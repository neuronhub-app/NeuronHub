## Desc

We're adding jobs landing pages with filters (by orgs, tags, etc), url, and SEO (title, desc, image, etc).

For example as "Climate Change jobs in France".

The filters must be visibly active and restable by the user.

The main long-term complexity may be in reliably converting Django filters to Algolia state.

## Decision-Log

#### v0 implementation
- kept prerender despite CLAUDE.md SSR ban — CTO: prefetch JSON + prerender ≠ unwanted SSR. Future refactors must not strip prerender to "comply" with the SSR ban.

#### drop complexity out of `client:prefetch-from-server`
- prefetch script logic → vite-node module (fetch + fs).

#### props.header → useHeroHeader hook
- self-cleanup via location-keyed state (`state.forPath`):
    - tried: PgLayout reset in render → infinite-loop (render-time mutation + subscription; `networkidle` timed out).
    - settled: writer records `location.pathname`; reader returns defaults on mismatch → SSR module-cache leak + client-nav self-clear w/o Layout reset.
- Valtio over React-native title: native emits 2 `<title>`/`<meta>` (defaults + override) → breaks Playwright strict locator + OG-first crawlers.

#### drop pytest API-coverage tests
- BE pytest = security; API shape coverage = e2e's job.

#### landing page Algolia URL params + reset redirect
- `routeToState({}) → preset` survives empty URL on first load AND reloads.
- equality via `JSON.stringify` — brittle: relies on key-order parity between `landingPageToAlgoliaState` output and Algolia roundtrip.
- `RedirectOnFiltersReset` kept local to `JobList` (child of `<PgAlgoliaList>`) — must live inside `<InstantSearch>` for `useClearRefinements` ctx (refactor trap).
- `wasActiveRef` in `RedirectOnFiltersReset`: skips canRefine=false window before preset is applied.

#### CR locked-in (no code change)
- FE prefetch `print(doc)` does NOT bypass BE whitelist — `PersistedQueryExtension.on_parse` asserts unconditionally; matches via normalized AST. `graphql.persisted()` does real work: gql.tada CLI whitelist + `ResultOf<typeof>` types.

#### CR-2: split prefetch — query / lookup / runner
- split avoids re-export footgun: prior `JobsLandingPages.ts` re-exported types from a module w/ top-level `await runPrefetch()`.

#### restore PgFacetPopover lazyMount
- `<RegisterLandingPageRefinements />` must live inside `<InstantSearch>` ctx (refactor trap).
- rejected: hoisting hooks into `PgFacetAttribute` — unrelated to feature.
- `NhaPosthogProvider`: gated `posthog.init` on `env.mode.isClient` — vite-node prerender stubs `posthog.default` (no `.init`) → SSR threw.
