### Requirements (clarified during session)

1. `id` prop must stay short — MDX has no TS autocomplete
2. `as ReactRouterPath` cast is forbidden — kills autocomplete in TS files
3. No codegen scripts — "you literally have ReactRouterPath"
4. Must not lose existing typing (`keyof typeof links`, `satisfies`)
5. The hardcoded `const links` should be replaced with something
   derived from existing code (navTree/routes)
6. Display title should come from navTree, not `props.id`
7. Support `hash` for `#section` anchors

### What remains unsolved

The `const links` in `PageLink.tsx` is still hardcoded. Every approach
to eliminate it was rejected.

The core tension: navTree produces strings at runtime, but
`ReactRouterPath` is a compile-time union. Bridging them without
cast or codegen was not solved. [it's created by typegen and placed in docs/.react-router/]

Possible approaches not yet tried:
- Template literal types extracting last segment from ReactRouterPath
- A typed helper that maps ReactRouterPath values to short ids at
  the type level
