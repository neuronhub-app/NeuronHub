## Desc

Our E2E runs `db_stubs_repopulate` before each test, which is too slow - total runtime on `VITE_SITE=""` with ~20 tests is closer to 10m, the `pg` site is 4-5m with ~3 tests.

We've added basic params to the GraphQL `test_db_stubs_repopulate` mutation, but they just barely reduce Algolia reindex time.

We've started splitting the sub-files as `jobs/tests/db_stubs.py`, but it was barely done only for `jobs`. And for `profiles` the LLM code should be completely rewritten.

Ideally the API of `test_gen` would be identical to the GraphQL in the end. But we can keep it as a long-term goal and start with incremental improvements.

There's no backward compatibility to account for.

The E2E doesn't need many co-dependant models (unlike pytests), and when it does - the ROI of it complexity needs to be confirmed.

## Decision-Log

- Bottleneck: ~50s/test = baseline seed + 3-index Algolia reindex.
    - HN import +3s.
    - 0/13 E2E specs need cross-ref of created entities.
- Chose tagged-union `GenCall` (nullable field per `Gen` method) + ordered `list[GenCall]`. Rejected:
    - per-method Strawberry mutations: schema bloat
    - scenarios registry of named bundles: magic strings
    - JSON dispatcher: untyped trash
    - exec(python_source): dumb
    - postgres template DB: dumb
    - autogen Strawberry from `Gen`: zero-maint claim is dumb
- Refs as natural-key strings (`parent`, `author`) -> Avoids returning IDs to TS.
- Shared auth via PW Project dep + `use.storageState` at project level (still needed for unmigrated specs).
    - Tradeoff: edit default user leaks across e2e -> it's never edited.
    - `test_db_stubs_repopulate` flips `is_delete_user_default=False` deleted sessions don't break the cookie.
- Auth refactor: `e2e/test.ts` (`test`/`testNoAuth`).
    - `auth.setup.ts` kept PW-idiomatic.
    - `e2e/auth.state.json`.
- `test_gen` mutation (posts_tool, posts_review only).
    - KISS: skip rating/importance (unused in e2e).
    - vote-and-reading-list: ~50s → ~13s/test.
- Dropped `genResetAndLogin` (auth via fixture, not helper).
