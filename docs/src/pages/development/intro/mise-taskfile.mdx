`mise.toml` is the single source of truth of envs and configuration.

Mise is using Nushell for all tasks.

### Primary Tasks

- `mise pytest` - server/ pytest only
    - Note: by default it excludes `pytest --markers "not slow_llm_api and not firebase_subscription"`.
      If needed run `test:pytest:all`.
    - Note: The `slow_llm_api` marker fails if executed inside `claude`, because `claude` forbids child `claude` process.
- `mise e2e` - Playwright on `client/`.
    - It starts its own Django and Vite servers on each run.
    - At first only run test cases you need, as full suite takes 10+ minutes.
    - Never add flags as `headless`.
    - `devops/db.mise.toml` - contains `dev:db:e2e:{setup|rm}` - helps to fix corrupted e2e db.
    - `devops/e2e.mise.toml` - just `runserver`, `db_worker`, and `react-router dev`.
- `mise e2e:docs` - for `docs/e2e/`.
- `mise lint` runs:
    - `format` - Ruff, Biome
    - `typegen` - Strawberry → schema.graphql → TS enums; gql.tada Persisted Queries (aka Backend whitelist); @chakra-ui; etc
    - `lint:biome` - for `client/`, `docs/`, and `packages/shared/`
    - `lint:client`, `lint:docs`, `lint:packages` - tsgo
    - When `lint:` tasks drop their stdout, except stderr.
    - If you see errors re `pg-primary` or `pg-highlighted` -> run `touch mise.toml && mise lint` once. That's just a chakra theme type bug.
- `mise django:migrate`
- `mise django:makemigrations`
