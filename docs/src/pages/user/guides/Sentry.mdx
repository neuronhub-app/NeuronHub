## Issue Reports

I recommend triaging the `backend` issues daily, and the `frontend` once in a few weeks.

The <LinkExt href="https://sentry.io/issues/alerts/rules/">default Alerts settings</LinkExt> are configured by project:
- Uptime - immediate email.
- `backend` - immediate email.
- `frontend` - if it affects 3+ users per month.

## Feedback

On error, a user is presented with a low-key popup to submit the Feedback report form:

<LinkExt href="https://sentry.io/issues/feedback/">The reports</LinkExt> can be viewed for 90 days.

Each sends an email alert. You can also enable <LinkExt href="https://sentry.io/issues/feedback/">Slack or SMS notifications</LinkExt>.
